# TTS Jarvis Lab

Это изолированная песочница для проверки идеи: OpenClaw отвечает как обычно, а UI дополнительно проговаривает ответ вслух.

## Что уже подтверждено

- Репозиторий `Priler/jarvis` не дает нам готовый универсальный TTS-движок.
- Внутри него лежит набор коротких предзаписанных реплик и реакций (`run`, `reply1`, `ok1`, `thanks`), а не синтез произвольного текста.
- Для прямого переноса кода есть лицензионная неопределенность:
  - `README` и `LICENSE.txt`: `CC BY-NC-SA 4.0`
  - `Cargo.toml`: `GPL-3.0-only`

## Что сделано в этом этапе

- В `UI` добавлен экспериментальный клиентский режим `Jarvis TTS`.
- Когда приходит новый финальный ответ ассистента, UI может озвучить его через системный `speechSynthesis`.
- Режим включается кнопкой `Jarvis TTS` в нижнем доке.
- Это не "настоящий голос из репозитория Priler/jarvis", а первый рабочий контур для проверки UX.

## Зачем этот этап

Нам нужно быстро проверить базовый сценарий:

1. Пользователь пишет или говорит в UI.
2. OpenClaw отвечает как обычно.
3. UI проговаривает финальный ответ.
4. Мы оцениваем, удобно ли это в реальном потоке.

## Ограничения текущего прототипа

- Голос зависит от системных голосов Electron/macOS.
- Нет клон-голоса Jarvis.
- Нет аудио-стриминга токен-в-токен, только озвучка финального ответа.
- Нет отдельного серверного TTS-провайдера в этом лабе.

## Следующий реалистичный шаг

Если базовый UX понравится, дальше можно идти по одному из двух путей:

- `UI-side TTS v2`: заменить `speechSynthesis` на отдельный локальный или облачный TTS-провайдер с более "Jarvis-like" голосом.
- `Voice pack + TTS`: оставить фирменные короткие реакции как earcons или интро, а длинные ответы синтезировать отдельным TTS.

Второй путь ближе всего к тому, что визуально и эмоционально напоминает JARVIS, не ломая OpenClaw.
