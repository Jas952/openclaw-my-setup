# Knowledge Base

`core/knowledge-base/` — это локальная RAG knowledge base, которую я использую как долговременную исследовательскую память для бота.

## За что она отвечает

Этот модуль позволяет сохранять статьи, ссылки, PDF и текстовые материалы в локальную базу знаний, разбивать их на чанки, строить для них эмбеддинги и потом находить релевантный контекст по естественному запросу.

На практике я использую это так:

- в Telegram отправляю боту ссылку, статью или документ;
- материал автоматически или вручную сохраняется в knowledge base;
- текст режется на чанки и индексируется;
- позже я в любой момент могу спросить бота по этой теме, и он быстро поднимает нужный контекст из локальной памяти.

Отдельно важно: эта возможность выведена и в UI-дизайн через вкладку `Library`, где можно видеть сохраненные материалы и работать с ними как с локальной исследовательской библиотекой.

Ниже в README можно будет прикрепить скриншот того, как эта вкладка выглядит.

## Модель и векторный слой

Здесь используется локальная embedding-модель:

- модель: `Xenova/multilingual-e5-small`
- тип: multilingual embedding model
- размер вектора: `384`
- языки: хорошо подходит для RU + EN сценариев
- режим работы: полностью локально, без API-ключей

Почему это интересно:

- модель достаточно компактная для локальной работы;
- поддерживает мультиязычный поиск;
- подходит для semantic retrieval, а не только keyword match;
- работает оффлайн и не зависит от внешнего embedding API.

Векторное хранилище:

- `zvec` используется как локальный vector store;
- SQLite хранит записи, метаданные и чанки;
- векторы и метаданные используются вместе для последующего retrieval.

## Технические характеристики

По текущей конфигурации:

- `embeddingDim`: `384`
- `chunkSize`: `500`
- `chunkOverlap`: `50`
- database path: `~/.openclaw/knowledge-base/data/knowledge.db`
- collection metric: cosine similarity
- index type: flat

## Как это работает

```text
URL / PDF / текст / локальный файл
        │
        ▼
  ingest.js
        │
        ├── извлекает текст
        ├── режет на chunks
        ├── строит embeddings
        ├── сохраняет metadata в SQLite
        └── сохраняет vectors в zvec
                 │
                 ▼
            query.js
                 │
                 ├── строит embedding запроса
                 ├── делает semantic search
                 └── возвращает лучшие chunks
```

## Структура

```text
core/knowledge-base/
├── ingest.js      — добавление URL, PDF, текста и файлов в базу знаний
├── query.js       — semantic query по knowledge base
├── list.js        — список сохраненных материалов
├── delete.js      — удаление записи из базы
├── embed.js       — локальная embedding-логика
├── db.js          — SQLite + zvec слой
├── kb.config.json — параметры модели, chunking и dataDir
└── SKILL.md       — правила использования knowledge-base внутри OpenClaw
```

```text
1. [84.4% match] Article Title
   Source: https://example.com
   Tags:   finance, notes
   ---
   Excerpt from the matched chunk...
```

### При просмотре библиотеки

```text
Knowledge Base — 18 entries, 146 total chunks
```

## Чем это полезно

- бот не "забывает" полезные статьи после одного диалога;
- можно накапливать собственную исследовательскую базу;
- retrieval идет по смыслу, а не только по точным словам;
- это удобно работает и в Telegram, и через UI-вкладку `Library`.
