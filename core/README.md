# Core

`core/` — это внутренний прикладной слой моего OpenClaw workspace: здесь лежат сервисные модули, которые расширяют базовый OpenClaw и закрывают отдельные реальные сценарии эксплуатации.

## Как это связано с OpenClaw

На уровне OpenClaw это не "ядро upstream", а мой operational-слой поверх базовой инфраструктуры:

- хранение и поиск знаний;
- резервное копирование локальных данных;
- трекинг использования моделей;
- журналирование cron-процессов;
- rule-based постобработка ответов;
- отдельные UI/office-эксперименты.

## Структура

```text
core/
├── backup/              — зашифрованные бекапы локальных баз и данных
├── cron-log/            — журналирование и аудит cron-задач
├── humanizer/           — rule-based “очеловечивание” исходящих ответов
├── knowledge-base/      — локальная RAG knowledge base с векторным поиском
├── model-usage-tracker/ — usage/cost/volume трекинг по моделям и агентам
└── office-ui/           — отдельный office/presence UI-проект
```

## Как это работает

- `backup/` берет локальные БД, шифрует архивы и отправляет их в удаленное хранилище.
- `cron-log/` фиксирует старт, завершение и сбои периодических задач.
- `humanizer/` проходит по исходящему тексту и убирает шаблонные AI-паттерны без LLM.
- `knowledge-base/` сохраняет статьи, ссылки и документы в локальную базу чанков и векторных эмбеддингов.
- `model-usage-tracker/` собирает usage-статистику по вызовам моделей и строит сводные отчеты.
- `office-ui/` отвечает за отдельный визуальный слой присутствия и состояния агентов.

## Примеры результата

- knowledge-base позволяет позже задать вопрос по ранее сохраненной статье и получить релевантные чанки.
- backup дает восстановление локальных данных после потери среды.
- model-usage-tracker показывает, какие модели использовались, в каком объеме и каким агентом.
- cron-log помогает быстро понять, где и когда упала периодическая задача.
